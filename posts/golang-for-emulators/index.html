Golang for Emulators

July 1, 2025

I am putting the last touches on my GBA Emulator Guac ðŸ¥‘, and have many thoughts. Most emulators are written in C/C++, or Rust for obvious reasons - performance; however, I want to push back against this being the default for a few reasons, and complain about some aspects of low level languages that I do not understand. Why use golang? Performance.

Gripes I have with C++ / Rust

Macros

I’m sure this is a code styling decision, but I cannot understand C Macros for the life of me.

Typing

I prefer a simple typing system that limits expression with more rebose logic. Often times in C++ code I have to spend most of my time understanding complex types written to avoid a few extra lines of code per function. I found the most typing I did in golang was casting uint32’s to a specific type, to allow for specific methods. For example, certain IO Registers can be cleaner with attached methods, like DISPSTAT with a VCOUNTER setter
